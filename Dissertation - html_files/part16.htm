Chapter 4: Data Collection


Machine learning, much like most software development, requires data. Within this project, the data needed are images that can be used to train a model and test the results from that model. Ideally, the data collected for this project would be produced by attaching a camera system to a truck as it makes its recycling run. However, after months of trying and failing to access the needed images, a new different set was used.


Training the model requires many sample images, which allows the model to analyse multiple different scenarios and extract the relevant features. Within this project, the training images came from the trashnet project [43], in which a single piece of recycling or rubbish was laid on a white background and an image was taken in figure 1.




Figure 1: Example of an image used for training


Testing the results from the trained model required a different set of images, as the single images did not represent the real-world use case this project was trying to solve. A recycling bin was emptied onto a garage floor, and pieces of rubbish were added to specific images, as shown in figure 2. Once an image had been taken, the recycling and rubbish were shifted around to create a new image. Collecting the test data allowed the results to reflect a more real-world scenario.
