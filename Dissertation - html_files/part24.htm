VII: Testing

Once a model has been trained, testing needs to be run on the results to evaluate how the model performed. For testing, the test dataset was used.


Testing requires metrics against which the results must be tested to determine the accuracy of the model's predictions. IOU (Eq.1) (intersection over union) is a metric that measures the area of overlap vs the area of the union of pixels in the test images vs the predicted results. The formula for the IOU metric is


ùêºùëÇùëà = Area of Overlap. (Eq. 1)

Area of Union



Area of overlap is defined as the pixels that overlap in the same area for the predicted results vs the true positives. The area of union is the total pixels union area between predicted data vs the true positives. IOU shows us how accurate the model is and how far away the model is from the actual value. An IOU value of 1 is a perfect score. These IOU values can be changed into a percentage to gauge a better idea of the accuracy of the model.


The keras_seg script was utilised, and the evaluate_model function was used. This function requires a directory for checkpoints of the trained model and an image path for the images to predict. This

function then calculates the IOU value of the predicted values and produces the IOU score for analysis.
